# `manifest_gois.json`（schemaVersion 3）

## 正本

- ソース: `https://scp-jp.wikidot.com/goi-formats-jp` の `#page-content` 内、`.content-panel` 優先（無ければ `#page-content` 直下を走査）
- タグ一覧（`goi-format`）は **子記事列挙には使わない**（一覧に載るのは同ページ上の `ul` 内リンクのみ）

## 分類

| 区間 | 検出 | `goiRegions` キー |
|------|------|-------------------|
| 要注意団体-EN ブロック | `h1` 文言に `要注意団体-EN` を含む | `en` |
| 要注意団体-JP ブロック | 同上 `要注意団体-JP` | `jp` |
| 上記以外で `要注意団体-` で始まる `h1`（`インフォメーション` 除外） | 例: `要注意団体-CN` など。細分化はせず **1 束** | `other` |

## 団体（1 エントリ = 1 `h2`）

- 直前の兄弟 `<p><a name="...">` があれば、その `name` を団体 `i`（なければ `h2` 見出しから ASCII 仮 `id`）
- `h2 > span` に `a[href]` がある: 同一スラッグ（単一セグメント）の場合のみ `u`（団体ハブ）として保持。`class="newpage"` も同様
- リンクが **ない** 場合: `t`（団体名）のみ。`u` キーは **省略**（未作成・テキストのみの団体）
- 子記事: 当該 `h2` の次の兄弟を走査し、次の `h1` / `h2` 手前までの `ul`（ネスト可）内の、同一 Wiki 上の **単一スラッグ**への `a` を **表示順**で列挙（`entries`）
- 外部サイトのみ・`system:` 等は従来どおり除外

## トップレベル

- `schemaVersion`: `3`（他マニフェストの `2` とは独立）
- `entries`: 全子記事のフラット配列（検索・互換。各 `i` について `metadata[i].g` に団体名、`metadata[i].r` に `en` / `jp` / `other`）
- `goiRegions`: `{ "en" | "jp" | "other": [ 団体オブジェクト, ... ] }`
- 団体オブジェクト: `{ "i", "t", "u"?, "entries": [ { "u", "i", "t" }, ... ] }`

## 互換

- 旧 **schema 2**（`goiRegions` なしのフラットのみ）は、クライアントが従来 UI にフォールバック可能。
